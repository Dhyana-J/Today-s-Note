단방향 데이터 통로! (Input / Output)
각각의 장치마다 연결할 수 있는 스트림이 있다. 
입출력을 수행하기 위해서는 스트림이 2개가 필요하다. (입력 하나 출력 하나)


* 외부 매체와의 연결성에 따라
 - 기반(기본 스트림) : 외부 매체와 직접적으로 연결하는 통로
 - 보조 스트림 : 단독으로는 사용할 수 없고, 외부 매체 사이에서 보조역할해주는 통로.


* 통로의 사이즈에 따라 (I/O)
 - 바이트(byte) 스트림 : 1byte (InputStream / OutputStream)
 - 문자(char)스트림 : 2byte (Reader / Writer)


// "바이트 기반 스트림" 가지고 데이터 입출력 할 것!!
// "바이트 스트림" : 데이터를 1바이트 단위로 밖에 전송이 불가한 통로.. 1바이트는 한글 깨진다. (2byte이상이어야함)
// "기반 스트림" : 외부 매체와 직접적 연결되는 핵심통로!
		
// "바이트 기반 스트림" --> 외부매체를 지정하고 그 외부매체와 직접적으로 연결된 통로에 데이터를 1바이트 단위로 전송!
		
// XXXInputStream : XXX매체로부터 데이터 입력받는 통로
// XXXOutputStream : XXX매체로 데이터 출력하는 통로
		
// FileInput(Output)Stream : 외부매체의 파일
// Audio ~ : 외부매체의 오디오
// Piped ~ : 외부매체의 또 다른 프로그램
